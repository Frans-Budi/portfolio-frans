Refactor the Core Stack section on the home page to make it feel more structured, more premium, and less visually flat.

Goals:

- Turn the section from a plain grouped text list into a stronger capability-driven UI
- Improve hierarchy, scanability, and visual depth
- Better communicate my main strengths, not just a list of technologies
- Keep the section clean, professional, and modern

Current issue:
The Core Stack section feels too flat because each category is presented as a simple heading followed by plain inline text. It looks like a grouped keyword list instead of a polished product-oriented skill presentation.

Required improvements:

1. Convert each stack group into a dedicated card or surface block.
2. Add clear visual hierarchy between primary and secondary groups.
3. Use Lucide icons for each stack category.
4. Add a short one-line descriptor under each category title.
5. Render technologies as chips / pills / tags instead of plain inline text.
6. Improve spacing, card contrast, and section rhythm so the section is easier to scan.
7. Preserve the existing categories, but present them in a more intentional and visually engaging way.
8. Add AI tools into the Tools & Workflow category: Codex, GitHub Copilot, and ChatGPT.

Recommended category icon mapping:

- Mobile Development → Smartphone
- Backend & Web → ServerCog or Globe
- Payments & Fintech → Wallet or CreditCard
- Databases & Services → Database
- Tools & Workflow → Wrench or Workflow

Hierarchy guidance:

- Make Mobile Development the strongest visual anchor
- Give Backend & Web and Payments & Fintech strong secondary emphasis
- Keep Databases & Services and Tools & Workflow slightly quieter

Chip/logo guidance:

- Only use logos for recognizable branded tools/technologies
- Keep concept-based items as text-only chips
- Logos must remain visually subtle and consistent in size
- Avoid making the chips look noisy or overly decorative
- Use logos to improve scanability, not to overwhelm the layout

Design rules:
