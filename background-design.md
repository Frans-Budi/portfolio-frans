Refactor the background system of my portfolio website to make it feel more consistent, polished, and premium.

Main objective:
Create a cohesive visual background language across the entire site. The current backgrounds feel inconsistent between sections, so I want a more deliberate system that improves scroll rhythm, hierarchy, and overall visual harmony without making the website feel noisy or overly decorative.

Design direction:
Use a dark neutral base with soft ambient cyan glow accents. The overall look should feel modern, technical, clean, and professional — suitable for a developer portfolio focused on real-world products. Avoid flashy gradients, random section colors, or heavy visual effects. The background should support the content, not compete with it.

Background system requirements:

- Use one primary global background color for the whole site.
- Use one secondary section background tone for subtle alternation between sections.
- Use one main accent effect only: soft cyan ambient glow.
- Use one decorative motif only, and keep it extremely subtle.
- Keep 80% of the page visually clean and flat.
- Use decorative background effects only where they add hierarchy.

Section styling guidance:

- Hero section:
  Make this the most visually rich section.
  Add a soft radial cyan glow behind the main visual or image area.
  Optionally include a very subtle grid or technical pattern, but keep it low contrast.
  This section should feel premium and attention-grabbing without looking busy.

- About section:
  Reduce visual effects significantly.
  Keep the background mostly clean and calm.
  This section should act as a visual resting point after the hero.

- Projects section:
  Keep the background mostly flat so the project cards stand out clearly.
  Use subtle contrast from the page background if needed, but do not add strong effects.

- Stack / Skills section:
  Reintroduce a very subtle technical motif such as a faint grid or dot pattern.
  Keep it understated and consistent with the hero, not stronger.

- Experience section:
  Return to a simple and quiet background.
  Prioritize readability and structure.

- Demo / Showcase section:
  Allow a small amount of ambient glow if needed, but keep it restrained.

- Achievements / Stats section:
  Keep this one very clean and minimal.

- Final CTA section:
  Give this section stronger visual emphasis than normal sections, but still within the same design system.
  Use a slightly richer background treatment so it feels like a strong ending.

Visual rules:

- Do not use a different background style for every section.
- Do not mix too many gradients, patterns, overlays, or colors.
- Do not make the backgrounds brighter or more complex than the cards/content.
- Keep all effects soft, blurred, and low-contrast.
- Maintain strong readability and content focus.

Implementation guidance:

- Reuse existing design tokens if possible.
- If needed, create reusable background utility classes or wrapper components for section backgrounds.
- Keep the implementation maintainable and scalable.
- Prefer subtle layered backgrounds using radial gradients, low-opacity overlays, and minimal patterns.
- Make sure the result looks good on both desktop and mobile.
- Preserve the current content structure and layout; focus only on improving the background system and visual consistency.

Expected outcome:
The website should feel more unified, more intentional, and more premium. Scrolling from section to section should feel smooth and structured, with clear visual rhythm. The final result should look like a modern product-focused developer portfolio, not a template with random decorative backgrounds.
