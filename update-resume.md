Refactor the Resume page to feel more premium, recruiter-friendly, and visually structured, especially on desktop.

Main objective:
Turn the current Resume page from a flat single-flow information page into a stronger recruiter-facing experience with better hierarchy, better scanability, and more intentional use of layout.

Current issues:

- The Resume page feels too flat and too linear, especially on desktop.
- The content is readable, but it looks more like a plain text summary than a polished resume experience.
- Experience, education, and skills all have similar visual weight, so the page lacks hierarchy.
- The page does not fully reflect the depth of the actual CV/resume content.
- Desktop space is underused.
- The Download CV action exists, but it does not feel prominent enough.

Goals:

- Improve desktop layout significantly while keeping mobile clean and readable
- Make the page easier for recruiters and HR to scan in a few seconds
- Increase visual depth without making the page feel noisy
- Better highlight experience, education, credentials, and strongest technical capabilities
- Preserve the current professional tone of the portfolio

Required improvements:

1. Redesign the top of the Resume page into a stronger “resume hero” section with:
   - name
   - role
   - short summary
   - quick credibility facts
   - clear CTA actions such as Download Resume PDF and Contact

2. Improve desktop layout by using the available horizontal space more intentionally.
   Consider a 2-column structure or a main-content + sidebar layout.
   The desktop page should not feel like a long flat single column.

3. Refactor the experience section into stronger timeline cards or structured experience blocks.
   Each item should emphasize impact, not just role and company.

4. Separate “Professional Experience” from “Education & Credentials” more clearly.

5. Compress education and course content into stronger credibility highlights instead of long plain lists.
   Prioritize items such as:
   - Bangkit Academy 2024
   - Digital Business degree / GPA 3.95
   - selected Android / Kotlin certifications
   - startup / incubator achievements when relevant

6. Refactor the technical skills section so it feels more visual and capability-driven.
   Use grouped cards, tags, chips, or categorized modules instead of plain stacked lists.

7. Make the Download CV action more prominent.
   Keep the existing functionality, but improve its visibility and importance in the layout.
   Consider surfacing it both near the top and again near the end of the page.

8. Add stronger visual hierarchy across the page:
   - clearer section separation
   - stronger card surfaces
   - better spacing rhythm
   - improved contrast between content groups
   - more intentional typography scale

9. Keep mobile layout clean and readable, but prioritize improving the desktop presentation where the page currently feels weakest.

10. Do not make the page look like a generic SaaS marketing layout.
    It should still feel like a polished, modern developer resume page.

Content direction:

- Emphasize recruiter-relevant clarity
- Highlight real product work across freelance, internship, startup, fintech, and education contexts
- Surface concrete credibility signals where possible
- Make the page easier to scan in 5–10 seconds

Expected outcome:
The Resume page should feel more polished, more structured, and more persuasive. On desktop, it should no longer feel flat or under-designed. It should present Frans as a serious Flutter-focused developer with real-world product experience, startup perspective, and strong learning credentials.
